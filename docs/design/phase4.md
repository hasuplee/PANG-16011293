# Phase 4 설계 - 풍선 분열 규칙

관련 문서: [../PLAN.md](../PLAN.md) (Phase 4 항목), [../FEATURES/game_rule.md](../FEATURES/game_rule.md)

## 목표

Phase 3에서 서로 그냥 통과하던 총알과 풍선 사이에 충돌 판정을 추가한다. 대형 풍선을 맞히면 중형 2개로, 중형을 맞히면 소형 2개로 분열하고, 소형을 맞히면 분열 없이 소멸한다.

> **갱신**: 발사 방식이 즉시 도달형 와이어에서 총알이 날아가는 방식으로 바뀌면서([phase3.md](./phase3.md) 참고), 충돌 판정도 "한 프레임 동안 총알이 이동한 구간(직전 위치 ~ 현재 위치)과 풍선의 겹침 여부"를 검사하는 방식으로 설계했다. 이렇게 하면 총알이 빠르게 움직여 한 프레임을 건너뛰어도 판정이 누락되지 않는다(스윕 충돌 검사).

## 풍선 크기 단계

- `large` → `medium` → `small` 3단계. 각 단계는 반지름이 다르다 (`BALLOON_LARGE_RADIUS` > `BALLOON_MEDIUM_RADIUS` > `BALLOON_SMALL_RADIUS`).
- 풍선 객체에 `tier` 필드를 추가해 현재 어떤 단계인지 구분한다.

## 충돌 판정

- 총알은 발사된 x좌표에 고정된 채 매 프레임 위쪽으로 이동한다 (Phase 3과 동일).
- 매 프레임, 이번 프레임에 총알이 이동한 구간(직전 y ~ 현재 y)과 각 풍선의 x/y 범위가 겹치는지 검사한다.
  - 수평: 풍선 중심 x좌표와 총알 x좌표의 차이가 `풍선 반지름 + 총알 반지름` 이하
  - 수직: 총알이 이번 프레임에 지나간 구간이 `풍선 중심 y ± 풍선 반지름` 범위와 겹침
  - 두 조건을 모두 만족해야 충돌로 판정한다.
- 이렇게 프레임 간 이동 구간 전체를 검사하므로, 총알 속도가 빠르거나 프레임이 드문 상황에서도 풍선을 그냥 지나치는 일(터널링) 없이 항상 충돌이 감지된다.
- 한 번의 발사(총알 1개)는 동시에 여러 풍선을 맞히지 않고, 그 프레임에서 가장 먼저 판정된 풍선 1개만 맞힌다. 충돌이 발생하면 총알은 즉시 사라진다.

## 분열 처리

- `large` 풍선 충돌 시: 충돌 위치에서 `medium` 풍선 2개가 생성되며, 각각 반대 방향(좌/우)의 수평 속도와 위로 튀어 오르는 수직 속도를 갖고 갈라진다.
- `medium` 풍선 충돌 시: 같은 방식으로 `small` 풍선 2개로 분열한다.
- `small` 풍선 충돌 시: 분열 없이 배열에서 제거된다(소멸).
- 분열된 풍선들도 기존과 동일한 중력/바운드/벽 반사 물리 규칙을 그대로 적용받는다.

## 파일 구조

```
src/game/
  constants.ts   - BALLOON_MEDIUM_RADIUS, BALLOON_SMALL_RADIUS, BALLOON_SPLIT_SPEED_X/Y 추가
  balloon.ts     - Balloon 타입(tier 포함), 물리 갱신 함수, 스윕 충돌 판정(isHitByBullet), 분열 함수 등 순수 로직 분리
  useGameplay.ts - 발사 입력 처리 + 총알 이동 + 풍선 물리 갱신 + 충돌/분열 처리를 하나의 루프에서 담당하는 훅
```

- 총알과 풍선이 같은 프레임에서 서로의 최신 위치를 참조해 충돌을 판정해야 하므로, 발사 로직(기존 `useWire.ts`)과 풍선 로직(기존 `useBalloons.ts`)을 `useGameplay.ts` 하나로 합쳤다.
- `BalloonSprite.tsx`(Windows 파일시스템 대소문자 충돌 회피를 위해 `Balloon.tsx`에서 이름 변경), `Bullet.tsx`(발사 방식 변경에 따라 `Wire.tsx`에서 이름 변경) 등 렌더링 컴포넌트는 `GameScreen`에서 풍선 배열을 순회하며 여러 개를 렌더링하도록 구성한다.

## Phase 4 테스트 포인트와의 매핑

- "대형 → 중형 → 소형 순서로 정확히 분열되는지" → `tier` 기반 분열 로직
- "분열된 풍선 2개가 서로 반대 방향으로 튀어나가는지" → 분열 시 좌/우 반대 수평 속도 부여
- "소형 풍선을 맞혔을 때 완전히 사라지는지" → `small` 풍선은 자식 없이 배열에서 제거

## 범위에서 제외한 것

- 플레이어와 풍선의 충돌(라이프 감소) - Phase 5
- 점수 획득, 스테이지 클리어 조건 - Phase 6
