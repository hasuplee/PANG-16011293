# Phase 3 설계 - 발사 및 풍선 이동

관련 문서: [../PLAN.md](../PLAN.md) (Phase 3 항목), [../FEATURES/game_rule.md](../FEATURES/game_rule.md)

## 목표

발사 키를 누르면 플레이어 위치에서 위쪽으로 총알이 날아가는 연출을 추가하고, 대형 풍선 1개가 중력을 받아 바운드하며 벽에 부딪히면 반사되도록 만든다. **이 단계에서는 총알과 풍선 사이의 충돌 판정은 구현하지 않는다** (서로 그냥 통과, Phase 4에서 분열 규칙과 함께 구현).

> **갱신**: 최초 설계는 발사 즉시 화면 상단까지 도달하는 "즉시 도달형 와이어"였으나, 실제 총이 발사되는 듯한 시각적 모션을 원하는 피드백에 따라 총알이 일정 속도로 눈에 보이게 이동하는 방식으로 변경했다 (아래 내용은 변경된 설계 기준).

## 발사(총알)

- **발사 키**: Space bar.
- 발사 시 그 순간의 플레이어 중심 x좌표에서 총알이 생성되며, 매 프레임 `BULLET_SPEED`(px/초) 속도로 위쪽으로 이동한다.
- 총알이 뷰포트 상단에 도달하거나 풍선에 맞으면 사라진다.
- 총알이 아직 사라지지 않은 상태에서는 재발사가 되지 않는다 (원작의 "무기 1개" 동작 방식과 동일 - 동시에 여러 발을 쏘지 않음).
- 총알은 발사된 x좌표에 고정된 채 수직으로만 이동하며, 발사 이후 플레이어가 이동해도 따라가지 않는다.

## 풍선 물리

- 대형 풍선 1개가 스테이지 상단 근처에서 시작한다.
- 매 프레임 중력 가속도(`GRAVITY`)만큼 수직 속도(`vy`)가 증가한다.
- 위치는 `x += vx * dt`, `y += vy * dt`로 갱신한다 (dt: 프레임 간 경과 시간).
- **바닥 충돌**: 풍선 바닥이 지면에 닿으면, `vy`의 부호만 반전(같은 크기 유지)한다. 에너지 손실이 없으므로 튕겨 오르는 높이가 이전과 동일하게 유지된다 ([game_rule.md](../FEATURES/game_rule.md)의 "같은 높이만큼 올라간다" 규칙).
- **벽 충돌**: 좌/우 벽에 닿으면 `vx`의 부호를 반전한다.
- **천장 충돌**: 안전장치로 천장에 닿으면 `vy` 부호를 반전한다 (일반적으로 도달하지 않지만 방어적으로 처리).

## 파일 구조

```
src/game/
  constants.ts   - GRAVITY, BALLOON_LARGE_RADIUS, BALLOON_INITIAL_SPEED_X, BULLET_SPEED, BULLET_RADIUS, FIRE_KEY 등 추가
  balloon.ts     - 풍선 물리 갱신 함수 (중력 + 바운드 + 벽 반사)
  BalloonSprite.tsx - 풍선을 원(circle)으로 렌더링
  useGameplay.ts - 발사 키 입력, 총알 이동, 풍선 물리를 함께 관리하는 훅 (Phase 4에서 충돌 판정까지 통합)
  Bullet.tsx     - 주어진 x, y 좌표에 총알을 렌더링
```

- `GameScreen.tsx`는 `Player`, `BalloonSprite`, `Bullet`을 모두 `GameViewport` 안에 배치한다.

## Phase 3 테스트 포인트와의 매핑

- "발사 시 총알이 플레이어 위치에서 나가서 위로 이동하는지" → `Bullet` 컴포넌트, 매 프레임 `BULLET_SPEED`만큼 위치 갱신
- "풍선이 바닥에 닿을 때마다 이전과 같은 높이로 튀어 오르는지" → `vy` 반전(감쇠 없음)
- "풍선이 벽에 부딪혔을 때 반대 방향으로 잘 튕기는지" → `vx` 반전

## 범위에서 제외한 것

- 총알과 풍선의 충돌 판정, 분열 규칙 (Phase 4)
- 풍선 크기 단계(중형/소형) - 이번 Phase는 대형 풍선 1개만 존재
