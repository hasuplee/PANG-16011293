# Phase 3 설계 - 발사 및 풍선 이동

관련 문서: [../PLAN.md](../PLAN.md) (Phase 3 항목), [../FEATURES/game_rule.md](../FEATURES/game_rule.md)

## 목표

발사 키를 누르면 플레이어 위치에서 위쪽으로 무기(와이어)가 화면 상단까지 즉시 도달하는 연출을 추가하고, 대형 풍선 1개가 중력을 받아 바운드하며 벽에 부딪히면 반사되도록 만든다. **이 단계에서는 와이어와 풍선 사이의 충돌 판정은 구현하지 않는다** (서로 그냥 통과, Phase 4에서 분열 규칙과 함께 구현).

## 발사(와이어)

- **발사 키**: Space bar.
- 발사 시 그 순간의 플레이어 중심 x좌표에 고정된 세로선(와이어)이 플레이어 위치부터 뷰포트 상단까지 즉시 나타난다.
- 와이어는 짧은 시간(`WIRE_DURATION_MS`, 150ms) 동안 표시된 뒤 사라진다.
- 와이어가 아직 사라지지 않은 상태에서는 재발사가 되지 않는다 (원작의 "무기 1개" 동작 방식과 동일 - 동시에 여러 발을 쏘지 않음).
- 와이어는 발사된 x좌표에 고정되며, 발사 이후 플레이어가 이동해도 따라가지 않는다.

## 풍선 물리

- 대형 풍선 1개가 스테이지 상단 근처에서 시작한다.
- 매 프레임 중력 가속도(`GRAVITY`)만큼 수직 속도(`vy`)가 증가한다.
- 위치는 `x += vx * dt`, `y += vy * dt`로 갱신한다 (dt: 프레임 간 경과 시간).
- **바닥 충돌**: 풍선 바닥이 지면에 닿으면, `vy`의 부호만 반전(같은 크기 유지)한다. 에너지 손실이 없으므로 튕겨 오르는 높이가 이전과 동일하게 유지된다 ([game_rule.md](../FEATURES/game_rule.md)의 "같은 높이만큼 올라간다" 규칙).
- **벽 충돌**: 좌/우 벽에 닿으면 `vx`의 부호를 반전한다.
- **천장 충돌**: 안전장치로 천장에 닿으면 `vy` 부호를 반전한다 (일반적으로 도달하지 않지만 방어적으로 처리).

## 파일 구조

```
src/game/
  constants.ts          - GRAVITY, BALLOON_LARGE_RADIUS, BALLOON_INITIAL_SPEED_X, WIRE_DURATION_MS, FIRE_KEY 등 추가
  useBalloonPhysics.ts   - 단일 풍선의 위치/속도를 rAF 루프로 갱신하는 훅 (중력 + 바운드 + 벽 반사)
  Balloon.tsx            - 풍선을 원(circle)으로 렌더링
  useWire.ts             - 발사 키 입력을 감지해 와이어의 x좌표(또는 null)와 지속 시간을 관리하는 훅
  Wire.tsx               - 주어진 x좌표에 세로선을 렌더링 (활성 상태일 때만)
```

- `GameScreen.tsx`는 `Player`, `Balloon`, `Wire`를 모두 `GameViewport` 안에 배치한다.

## Phase 3 테스트 포인트와의 매핑

- "발사 시 와이어가 즉시 화면 상단까지 나가는지" → `Wire` 컴포넌트, 플레이어 x좌표에서 뷰포트 상단까지 즉시 렌더링
- "풍선이 바닥에 닿을 때마다 이전과 같은 높이로 튀어 오르는지" → `vy` 반전(감쇠 없음)
- "풍선이 벽에 부딪혔을 때 반대 방향으로 잘 튕기는지" → `vx` 반전

## 범위에서 제외한 것

- 와이어와 풍선의 충돌 판정, 분열 규칙 (Phase 4)
- 풍선 크기 단계(중형/소형) - 이번 Phase는 대형 풍선 1개만 존재
